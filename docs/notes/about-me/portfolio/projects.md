# 重点项目

这里收录的项目偏向“解决真实问题”，重点不在项目数量，而在它们是否来源于明确场景、是否真的被使用过、是否能体现我的工程判断。

## 面向联调效率的工具

### api_proxy_tool_ext

- 项目地址：[Jsmond2016/api_proxy_tool_ext](https://github.com/Jsmond2016/api_proxy_tool_ext)
- 场景：浏览器接口快速联调，适配 Apifox，服务于团队开发流程。
- 特点：偏业务实战，直接针对开发中的重复性问题做提效。

## 面向开发过程记录的工具

### coding-history

- 项目地址：[Jsmond2016/coding-history](https://github.com/Jsmond2016/coding-history)
- 场景：回顾每日提交、辅助工时回溯和加班数据记录。
- 特点：把代码记录转成可视化看板，偏向个人效率和过程量化。

## 面向个人生活管理的工具

### simple-ledger-app

- 项目地址：[Jsmond2016/simple-ledger-app](https://github.com/Jsmond2016/simple-ledger-app)
- 场景：个人在线记账，支持导入 Excel 并进行月底统计分析。
- 特点：体现我会把技术能力延伸到自己的真实生活场景中。

## 面向编辑器工作流的插件

### wepy-action-nave-vsc-plugin

- 项目地址：[Jsmond2016/wepy-action-nave-vsc-plugin](https://github.com/Jsmond2016/wepy-action-nave-vsc-plugin)
- 场景：在 VSCode、Cursor 中快速跳转 wepy 相关文件。
- 特点：偏开发工作流优化，针对历史项目技术栈提供提效能力。

### commit-prefix-helper

- 项目地址：[Jsmond2016/commit-prefix-helper](https://github.com/Jsmond2016/commit-prefix-helper)
- 场景：根据 Git 分支名称提取 scope，辅助生成规范的 commit message 前缀。
- 特点：项目小而实用，体现我对工程规范和开发体验的关注。
